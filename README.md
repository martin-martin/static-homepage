# Martin Breuss - Static Homepage

A static site built with Hugo and the Terminal theme, featuring a portfolio and a
"Writing & Teaching" blog hub that aggregates Real Python tutorials, Real Python
video courses, and earlier Medium writing.

## Features

- **Portfolio/Projects**: Showcase of 39 projects with images, descriptions, and
  GitHub links. Each project's date is the real GitHub repository creation date.
- **Blog hub** (`content/blog/`): a single, auto-generated section combining three
  sources, each shown with a source badge, publish date, description, and (for
  Real Python content) a star rating:
  - **Real Python tutorials** — discovered dynamically from the author's team page,
    with authorship verified per article.
  - **Real Python video courses** — Martin's instructed courses.
  - **Earlier Writing** — a curated Medium archive (2017–2018).
- **Auto-deployment**: GitHub Actions automatically builds and deploys on push to main

> The content-generation scripts use **only the Python standard library** — no
> third-party packages are required.

## Setup

### Prerequisites

- Hugo (extended version)
- Python 3.11+ (standard library only)

### Local Development

1. Clone the repository
2. Install Hugo theme (already included as submodule):
   ```bash
   git submodule update --init --recursive
   ```
3. Generate content (no Python dependencies needed):
   ```bash
   python3 migrate_projects.py    # Projects (dates from GitHub)
   python3 fetch_rp_articles.py   # Blog hub: tutorials + courses + Medium
   ```
4. Run Hugo server:
   ```bash
   hugo server
   ```
5. Visit `http://localhost:1313`

### Building

```bash
hugo --minify
```

The built site will be in the `public/` directory.

## Content Structure

- `content/blog/` - Blog hub, regenerated by `fetch_rp_articles.py` (tutorials,
  courses, Medium). `content/blog/_index.md` is the hand-written section intro and
  is preserved across regenerations.
- `content/projects/` - Portfolio projects (generated from `content/projects/projects.json`)
- `content/contact.md` - Contact page
- `static/` - Static assets (images, favicon, etc.)

### Data sources (committed)

- `content/projects/projects.json` - hand-maintained project list
- `project_repo_dates.json` - cache of GitHub repo creation dates (per project)
- `posts_stats.csv` - Real Python article stats snapshot (used for star ratings)
- `courses_stats.csv` - Real Python video course list + ratings (instructor=mbreuss)
- `medium_posts.json` - curated Medium archive (edit by hand to add/remove posts)
- `rp_meta_cache.json` - cache of scraped article/course descriptions + dates

## Deployment

The site is automatically deployed to GitHub Pages via GitHub Actions when you push to the `main` branch.

### Automated Blog Updates

The blog hub is kept current automatically:
- **On every push**: The deployment workflow runs `fetch_rp_articles.py` before building
- **Weekly (Sun 2 AM UTC)**: A scheduled workflow regenerates the blog hub and
  commits `content/blog/` + `rp_meta_cache.json` if anything changed

You can also manually trigger the update workflow from the GitHub Actions tab.

### Custom Domain

The site is configured for `martinbreuss.com`. The CNAME file is in `static/CNAME`.

To set up the custom domain:
1. Go to your GitHub repository Settings > Pages
2. Enter `martinbreuss.com` in the Custom domain field
3. Configure DNS records as instructed by GitHub

## Updating Content

### Adding a New Project

Edit `content/projects/projects.json` and add your project, then regenerate:

```bash
python3 migrate_projects.py
```

The project date comes from the GitHub repo's creation date (cached in
`project_repo_dates.json`). Projects without a real GitHub repo fall back to a
default date.

### Updating the Blog Hub

```bash
python3 fetch_rp_articles.py
```

The script (standard library only):
- **Tutorials**: discovers all tutorials linked from
  `https://realpython.com/team/mbreuss/`, then keeps only those whose JSON-LD
  `author` is "Martin Breuss" (so tutorials Martin only edited are excluded).
  Titles, publish dates and descriptions come from each page's JSON-LD / meta tags.
- **Courses**: read from `courses_stats.csv` (instructor=mbreuss); descriptions
  scraped from each course page.
- **Medium**: read from `medium_posts.json` (edit by hand to curate).
- Star ratings are merged in from `posts_stats.csv` / `courses_stats.csv`. Refresh
  those CSV exports to update ratings.
- Scraped metadata is cached in `rp_meta_cache.json`, so reruns only fetch pages
  they haven't seen before. `content/blog/` is fully regenerated each run (except
  `_index.md`).

## Content Generation Scripts

- `migrate_projects.py` - Converts `projects.json` to Hugo markdown, dating each
  project by its GitHub repo creation date.
- `fetch_rp_articles.py` - Generates the unified blog hub (tutorials + courses + Medium).
